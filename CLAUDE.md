# 知识库协作指南

这是一个以**学习为核心**的个人知识库。

## 核心理念

**入库不整理，出库整理**

- 存入时零门槛，随便扔
- 学习时才开始处理
- AI辅助出库整理和展示

---

## 目录结构

```
知识库/
├── 00Inbox/         ← 收件箱，随便扔
├── 05Temp/          ← 临时笔记，用完即删或转正
├── 10Learning/      ← 正在学习的内容
├── 20Notes/         ← 永久笔记
├── 30Outputs/       ← 展示内容
│   ├── 技术/
│   ├── 读书/
│   ├── 生活/
│   ├── 工作/
│   └── 项目/        ← 项目MOC
└── CLAUDE.md
```

### 流转规则

| 阶段 | 动作                | 位置变化 |
|-----|------|---------|
| 存入 | 扔进Inbox，不用想任何事    | → 00Inbox/ |
| 讨论 | 临时笔记，讨论记录         | → 05Temp/ |
| 学习 | 拖到Learning，加 #进行中 | 00Inbox/ → 10Learning/ |
| 学完 | 改 #已完成 ，喊AI整理     | 10Learning/ → 20Notes/ |
| 展示 | 决定展示，AI美化分类       | 20Notes/ → 30Outputs/ |
| 临时转正 | 讨论深入，值得保留         | 05Temp/ → 20Notes/ |

### 临时目录规则

- **用途**：存放临时笔记、讨论记录、待整理的碎片
- **生命周期**：用完即删，或转正为永久笔记
- **不同步**：临时内容不需要同步到 GitHub（可选）

## 标签体系

**固定3个，不新增：**

| 标签 | 用途 |
|-----|------|
| `#进行中` | 正在学习的内容 |
| `#已完成` | 学完的内容，触发AI整理 |
| `#存档` | 不再更新，但保留 |

**禁止**：
- 不添加 frontmatter 中的 `tags` 字段
- 不创建任何新的 `#标签`
- 分类靠文件名前缀和目录结构，不靠标签

## AI协作规则

### 学习阶段
- 解释概念、举例说明、回答问题
- 生成练习题检验学习成果

### 整理阶段（触发条件：用户说"学完了" + 标签改为 #已完成 ）
1. 读取笔记内容
2. 整理结构（提炼要点、优化表达）
3. 按类型重命名（加前缀）
4. 移动到 20Notes/
5. 发现可能相关的笔记，**询问用户是否加链接**
6. 用户确认后，建立双向链接
7. 询问用户是否要展示

### 展示阶段（触发条件：用户说"要展示"）
1. 美化内容、优化排版
2. 按类型分类到 30Outputs/技术/ 等目录
3. **自动更新**对应MOC索引

### 项目聚合（触发条件：用户主动喊）
1. 分析笔记之间的关联
2. 发现可以组成项目的笔记群
3. 生成项目MOC，放在 30Outputs/项目/

### 维护任务（触发条件：用户主动喊）
- Inbox清理：帮用户快速过一遍Inbox
- 孤岛发现：找出没有链接的笔记
- 学习报告：统计学了什么

## Obsidian CLI 规则（重要）

**必须使用 Obsidian CLI 操作笔记**，以保护双链结构。

**所有笔记操作（创建、读取、修改、移动）都必须通过 CLI 命令执行。**

**CLI 路径：** `E:\002应用安装\002Obsidian\Obsidian.com`

**如果 CLI 不可用：**
1. 立即停止操作
2. 提醒用户：「请打开 Obsidian 并确保 CLI 可用」
3. 等待用户确认后再继续

**常用命令：**
```bash
# 读取笔记
"E:/002应用安装/002Obsidian/Obsidian.com" read file="笔记名" vault="000个人知识库"

# 创建笔记
"E:/002应用安装/002Obsidian/Obsidian.com" create name="新笔记" content="# 内容" vault="000个人知识库"

# 追加内容
"E:/002应用安装/002Obsidian/Obsidian.com" append file="笔记名" content="追加内容" vault="000个人知识库"

# 搜索
"E:/002应用安装/002Obsidian/Obsidian.com" search query="搜索词" vault="000个人知识库"
```

## Git 同步规则

**仓库地址：** https://github.com/fangtang951210/Obsidian.git

**同步范围：** 全部内容，排除 `000置顶消息【禁止编辑】/`

**触发时机：**
- Write/Edit 工具操作后自动同步

**Commit 格式：**

- 新建笔记：`➕ 新建：{笔记名}`
- 展示内容：`✨ 展示：{内容名}`
- 其他操作：`📦 更新：{简要描述}`

## AI不应该做的

- 不要在存入阶段要求用户做决策
- 不要预设分类，分类从内容中长出来
- 不要代替用户学习
- 不要添加不存在的链接：添加 `[[链接]]` 前必须用 CLI 搜索确认笔记是否存在，没有相关的就不填，绝不凭空编造
- 不要同步 `000置顶消息【禁止编辑】/` 目录

## 输出风格

- 用简体中文
- 解释技术概念时用生活化类比
- 优先给出结论，再解释原因
- 长内容要有清晰的结构和标题
